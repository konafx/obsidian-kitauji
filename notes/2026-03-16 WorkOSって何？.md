### 🔍 WorkOS

**一言で**: B2B向け認証・ID管理基盤

**カテゴリ**: SaaS / IDaaS（Identity as a Service）

---

#### 概要
- **何をするか**: B2B SaaS アプリケーションにエンタープライズ対応の認証（SSO, SCIM, MFA, Magic Link, Passkeys）とユーザー管理機能を提供するプラットフォーム
- **開発元**: WorkOS, Inc.（企業）— CEO: Michael Grinich（元Nylas創業者）
- **設立**: 2019年頃から。2024年にAuthKit（認証UI）をリリースし、B2C寄りのユースケースにも対応を拡大
- **ライセンス**: プロプライエタリ（SDKはOSS: MIT）
- **主要顧客**: Cursor, Perplexity, Vercel, Plaid, Loom など

#### 嬉しさ（なぜ使うのか）
- **AuthKitで数行で美しいログイン画面が完成** — Passkeys, Social Login, Magic Link, MFA が即座に使える
- **SSO追加が後付けで簡単** — 最初はAuthKitだけで始めて、企業顧客が出たらSSO接続を追加するだけ
- **1M MAUまで無料** — 個人開発・スタートアップの初期段階でコストゼロ
- **TypeScript-first** — Next.js App Router に公式SDK（`@workos-inc/authkit-nextjs`）があり、型安全
- **Multi-tenancy対応** — Organization モデルが組み込み済みで、B2B SaaSの設計がスムーズ

#### 競合・代替サービス

| サービス | 特徴 | 料金感 | 比較ポイント |
|----------|------|--------|-------------|
| **Clerk** | B2C/フロントエンド特化、React UIコンポーネント充実 | ユーザー単価課金 | UIカスタマイズ性◎、B2Bは弱め |
| **Auth0 (Okta)** | 老舗フルIDaaS、カスタマイズ自由度高 | ティア制、スケール時に高額化 | 機能最多だが複雑・高コスト |
| **Supabase Auth** | OSS、PostgresベースのBaaS付属 | 無料枠あり、安価 | 軽量で個人開発向き、エンタープライズ機能薄 |

| **SuperTokens** | OSS、セルフホスト可 | 無料（セルフホスト） | 自前管理したい人向け |

#### 所感

Grokの分析がかなり的確。「B2B SaaSを作る可能性が少しでもあるなら最初からWorkOS」というのは合理的な判断。1M MAU無料でAuthKitの体験が良いので、実質ノーリスクで始められる。逆に純粋B2Cなら Clerk か Supabase Auth の方が軽い。

### TypeScript 実装例（Next.js App Router + AuthKit）

#### 1. インストール

```bash
npm install @workos-inc/authkit-nextjs
```

#### 2. 環境変数（`.env.local`）

```env
WORKOS_CLIENT_ID=client_xxxxxxxxxxxxxxxx
WORKOS_API_KEY=sk_test_xxxxxxxxxxxxxxxx
WORKOS_COOKIE_PASSWORD=your-32-char-minimum-secret-password-here
NEXT_PUBLIC_WORKOS_REDIRECT_URI=http://localhost:3000/callback
```

#### 3. Auth コールバック（`app/callback/route.ts`）

```typescript
import { handleAuth } from "@workos-inc/authkit-nextjs";

export const GET = handleAuth();
```

#### 4. Middleware（`middleware.ts`）

```typescript
import { authkitMiddleware } from "@workos-inc/authkit-nextjs";
import type { NextRequest, NextFetchEvent } from "next/server";

export async function middleware(req: NextRequest, event: NextFetchEvent) {
  return await authkitMiddleware()(req, event);
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
};
```

#### 5. ユーザー取得（Server Component）

```typescript
import { getUser } from "@workos-inc/authkit-nextjs";

export default async function DashboardPage() {
  const { user } = await getUser();

  if (!user) {
    return <div>ログインしてください</div>;
  }

  return (
    <div>
      <h1>ようこそ、{user.firstName}さん</h1>
      <p>Email: {user.email}</p>
    </div>
  );
}
```

#### 6. サインアウト（Server Action）

```typescript
"use server";

import { signOut } from "@workos-inc/authkit-nextjs";

export async function logout() {
  await signOut();
}
```

#### 7. カスタム認証フロー（パスワード認証の場合）

```typescript
"use server";

import "server-only";
import { redirect } from "next/navigation";
import { z } from "zod";
import WorkOS from "@workos-inc/node";

const workos = new WorkOS(process.env.WORKOS_API_KEY!);

const signInSchema = z.object({
  email: z.string().email(),
  password: z.string(),
});

export async function signIn(_: unknown, formData: FormData) {
  const { email, password } = signInSchema.parse({
    email: formData.get("email"),
    password: formData.get("password"),
  });

  const result = await workos.userManagement.authenticateWithPassword({
    clientId: process.env.WORKOS_CLIENT_ID!,
    email,
    password,
  });

  // セッションCookieを設定（iron-session等でシール）
  // ...

  redirect("/dashboard");
}
```
